## 📅 2020 年

### 1. `clamp()`, `min()`, `max()` — 流体值函数

CSS 终于有了**数学函数**，用于定义响应式值而无需媒体查询。

```css
/* clamp(MIN, PREFERRED, MAX) - 最常用 */
font-size: clamp(1rem, 2.5vw, 3rem);

/* 响应式内边距 */
padding: clamp(1rem, 5%, 3rem);

/* min/max */
width: min(100%, 800px);      /* 取较小值 = 不超过 800px */
height: max(300px, 50vh);     /* 取较大值 = 至少 300px */
```

**适用场景**：响应式字体、弹性布局约束、自适应容器宽度、网格列数动态控制。

---

### 2. 逻辑属性（Logical Properties）

用 inline/block 方向代替物理方向（left/right/top/bottom），天然支持 RTL 书写方向。

```css
/* 以前 */
margin-left: 1rem;
padding-right: 2rem;
border-left: 1px solid;

/* 现代 - 自动适配书写方向 */
margin-inline-end: 1rem;
padding-inline-end: 2rem;
border-inline-start: 1px solid;
margin-block: 1rem 2rem;     /* margin-top + margin-bottom */

/* 定位 */
inset: 0;                      /* top:0; right:0; bottom:0; left:0 */
inset-inline: 1rem 2rem;       /* left:1rem; right:2rem (LTR) */
inset-block: 1rem;             /* top:1rem; bottom:1rem */
```

**适用场景**：多语言网站（阿拉伯语 RTL）、国际化应用、通用组件库。

## 📅 2021 年

### 3. `aspect-ratio` — 宽高比锁定

告别古老的 padding-bottom hack。

```css
.card-image {
  aspect-ratio: 16 / 9;
  object-fit: cover;   /* 配合裁剪 */
}

/* 正方形头像 */
.avatar {
  aspect-ratio: 1;
  border-radius: 50%;
}

/* 响应式视频容器 */
video {
  width: 100%;
  aspect-ratio: 16 / 9;
}
```

**适用场景**：图片/视频占位、卡片封面、图标容器、自适应媒体。

---

### 4. Flexbox `gap` — 弹性容器间距

```css
.flex-row {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;          /* 替代 margin + :last-child 清除 */
}

.grid-cards {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 1.5rem 2rem;   /* row-gap column-gap 分开设置 */
}
```

**适用场景**：任何 Flex/Grid 布局需要间距时，不再需要 margin hack。

---

### 5. `accent-color` — 表单控件一键主题色

```css
:root {
  accent-color: #6366f1;   /* 一次设置，影响所有表单控件 */
}

/* 也可单独设置 */
input[type="checkbox"] { accent-color: #22c55e; }
input[type="radio"] { accent-color: #f59e0b; }
input[type="range"] { accent-color: #ef4444; }
progress { accent-color: #3b82f6; }
```

**适用场景**：Checkbox、Radio、Range、Progress 的主题色统一。

---

### 6. `:is()` 和 `:where()` — 智能选择器

```css
/* :is() - 选择器列表，权重取最高选择器的权重 */
article :is(h1, h2, h3, .special) { margin-top: 2rem; }
/* 等价于 article h1, article h2, article h3, article .special */

/* :where() - 选择器列表，权重永远为 0 */
:where(.card, .panel, .box) :where(.title, .heading) {
  font-size: 1.25rem;
}

/* 典型用法：重置样式且不增加权重 */
:where(h1, h2, h3, h4) { margin: 0; }
```

**适用场景**：简化长选择器列表、第三方样式库重置（`:where()`）、组件样式覆盖。

## 📅 2022 年

### 7. `:has()` — 父级选择器（CSS 革命）

等了 20 多年的"父级选择器"，可以根据子元素的状态来样式化父元素。

```css
/* ⭐ 核心用法：包含图片的卡片切换布局 */
.card:has(img) {
  grid-template-columns: 1fr 2fr;
}

/* ⭐ 表单验证反馈 */
.form-group:has(:invalid) { border-color: red; }
.form-group:has(:focus) { box-shadow: 0 0 0 2px blue; }

/* ⭐ 自定义 checkbox 状态 */
label:has(input:checked) { font-weight: bold; }

/* ⭐ 空状态检测 */
.container:not(:has(> *)) { display: none; }

/* ⭐ 表格行高亮 */
tr:has(.starred) { background: #fef9c3; }

/* ⭐ 兄弟元素检测 */
.card:has(+ .card) { border-bottom: 1px solid #ddd; }
```

**适用场景**：表单验证反馈、卡片媒体检测、动态布局切换、表格行高亮、空状态处理。

---

### 8. `@layer` — 级联层（解决优先级战争）

```css
/* 定义层叠顺序（越后权重越高） */
@layer reset, base, components, utilities, overrides;

/* 框架层权重最低 */
@layer reset {
  *, *::before, *::after { margin: 0; padding: 0; box-sizing: border-box; }
}

@layer base {
  body { font-family: system-ui; line-height: 1.6; }
}

@layer components {
  .btn { background: blue; color: white; padding: 0.5em 1em; }
  .btn-danger { background: red; }
}

/* utilities 层权重最高 */
@layer utilities {
  .mt-4 { margin-top: 2rem; }
}

/* 覆盖第三方库 */
@layer overrides {
  .third-party-widget { color: inherit !important; }
}
```

**核心规则**：层顺序决定优先级 > 选择器权重。后定义的层始终高于前面的层。

**适用场景**：大型项目 CSS 架构、第三方库样式覆盖、设计系统分层管理。

---

### 9. `@property` — 类型化 CSS 自定义属性

让自定义属性可动画、可继承、可参与计算。

```css
/* ✅ 渐变角度动画 */
@property --gradient-angle {
  syntax: "<angle>";
  inherits: false;
  initial-value: 0deg;
}

.box {
  background: linear-gradient(var(--gradient-angle), #f00, #00f);
  transition: --gradient-angle 1s ease;
}
.box:hover { --gradient-angle: 180deg; }

/* ✅ 颜色动画 */
@property --brand-color {
  syntax: "<color>";
  inherits: true;
  initial-value: #6366f1;
}

.themed-box {
  background: var(--brand-color);
  transition: --brand-color 0.5s;
}
```

**适用场景**：渐变动画、颜色过渡、数字属性动画、Web 组件主题。

## 📅 2023 年

### 10. 容器查询（Container Queries） — 组件级响应式

CSS 最重大的突破之一。基于**父容器**的大小（而非视口）来做响应式。

```css
/* 第一步：定义容器 */
.card-container {
  container-type: inline-size;
  container-name: card;
}

/* 第二步：基于容器大小查询 */
@container card (min-width: 400px) {
  .card-inner {
    display: grid;
    grid-template-columns: 1fr 2fr;
  }
  .card-image { border-radius: 8px 0 0 8px; }
}

@container card (max-width: 399px) {
  .card-inner {
    display: flex;
    flex-direction: column;
  }
  .card-image { border-radius: 8px 8px 0 0; }
}

/* 容器查询单位 */
.card-title { font-size: clamp(1rem, 5cqi, 2rem); }
/* 5cqi = 容器 inline-size 的 5% */
```

**适用场景**：通用组件（无论放在侧边栏还是主区域都能自适应）、Dashboard 组件、复杂表单、卡片。

### 11. CSS 原生嵌套（CSS Nesting）

不再需要 Sass/Less。

```css
.card {
  background: white;
  border-radius: 8px;
  padding: 1.5rem;

  & .title { font-size: 1.25rem; font-weight: 600; }
  & .body { color: #666; line-height: 1.6; }
  & .footer { border-top: 1px solid #eee; padding-top: 1rem; }

  &:hover {
    box-shadow: 0 4px 12px rgba(0,0,0,.1);
  }

  /* 嵌套伪元素 */
  &::before { content: "★"; color: gold; }

  /* 嵌套媒体查询 */
  @media (max-width: 600px) {
    & { border-radius: 0; padding: 1rem; }
  }
}
```

**适用场景**：组件样式编写、替代 Sass 嵌套、简化选择器重复。

### 12. Subgrid — 继承父级网格轨道

```css
.parent {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
  gap: 1rem;
}

.child {
  display: grid;
  grid-column: 1 / -1;
  grid-template-columns: subgrid;  /* 继承父级的列 */
  gap: inherit;
}

/* 实际案例：对齐卡片内的元素 */
.card {
  display: grid;
  grid-template-rows: subgrid;    /* 所有卡片行高对齐 */
  grid-row: span 3;               /* 跨越 3 行 */
}
.card-image { grid-row: 1; }
.card-title { grid-row: 2; }
.card-body { grid-row: 3; }
```

**适用场景**：列表项内容对齐、表格类布局、多卡片统—行高。

### 13. OKLCH / `color-mix()` — 现代颜色函数

```css
:root {
  --brand: oklch(55% 0.23 280);  /* 感知均匀色空间 */
}

/* color-mix() - 混合颜色 */
--brand-hover:  color-mix(in oklch, var(--brand), black 15%);
--brand-active: color-mix(in oklch, var(--brand), black 30%);
--brand-bg:     color-mix(in oklch, var(--brand), white 80%);
--brand-border: color-mix(in srgb, var(--brand), white 40%);

/* OKLCH 相对颜色（2025 广泛支持） */
--brand-light: oklch(from var(--brand) 0.7 0.1 h);
--brand-dark:  oklch(from var(--brand) 0.3 0.15 h);

/* 实际使用 */
.btn {
  background: var(--brand);
  color: white;
}
.btn:hover { background: var(--brand-hover); }
```

**适用场景**：设计系统色板生成、主题切换、hover/active 派生色、无障碍对比度管理。

### 14. View Transitions API（视图过渡）

```css
/* SPA 页面过渡 - 全局淡入淡出 */
::view-transition-old(root) {
  animation: 0.3s ease-out both fade-out;
}
::view-transition-new(root) {
  animation: 0.3s ease-in both fade-in;
}

@keyframes fade-out { to { opacity: 0; } }
@keyframes fade-in { from { opacity: 0; } }

/* MPA 跨页面导航过渡 (2025 稳定) */
@view-transition { navigation: auto; }

/* 特定元素过渡 */
.product-image { view-transition-name: product-img; }
```

**适用场景**：页面切换动画、图片展开/收缩过渡、SPA 路由动画、MPA 页面间无缝过渡。

### 15. `scroll-snap` — 原生滚动吸附

```css
/* 水平轮播 */
.carousel {
  display: flex;
  overflow-x: auto;
  scroll-snap-type: x mandatory;
  scroll-behavior: smooth;
}

.slide {
  scroll-snap-align: center;
  flex: 0 0 100%;
}

/* 垂直分页 */
.pages {
  overflow-y: scroll;
  scroll-snap-type: y mandatory;
  height: 100vh;
}
.page {
  scroll-snap-align: start;
  height: 100vh;
}

/* 间距吸附 */
.gallery {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  overflow-x: auto;
  scroll-snap-type: x proximity;
  gap: 1rem;
}
.gallery-item {
  scroll-snap-align: start;
}
```

**适用场景**：轮播图、分页滚动、画廊、全屏滚动页面。

## 📅 2024 年

### 16. `text-wrap: balance` / `pretty` — 完美文本排版

```css
/* balance - 各行宽度均衡（适合标题） */
h1, h2, h3, .card-title {
  text-wrap: balance;
  max-width: 30ch;   /* 配合宽度限制效果更佳 */
}

/* pretty - 防止 orphan（孤行），适合段落 */
article p, .description {
  text-wrap: pretty;
}

/* 结合使用 */
.headline {
  text-wrap: balance;
  text-align: center;
}
```

**适用场景**：标题排版优化、卡片文字、文章正文段落、引用块。

### 17. `@starting-style` — 从 display:none 进入动画

```css
/* 弹出菜单 */
.menu {
  display: none;
  opacity: 0;
  translate: 0 -10px;
  transition:
    display 0.3s allow-discrete,
    opacity 0.3s,
    translate 0.3s;
}

.menu.open {
  display: block;
  opacity: 1;
  translate: 0 0;

  /* 进入动画的初始状态 */
  @starting-style {
    opacity: 0;
    translate: 0 -10px;
  }
}

/* Dialog 动画 */
dialog {
  opacity: 0;
  scale: 0.9;
  transition: opacity 0.3s, scale 0.3s, display 0.3s allow-discrete, overlay 0.3s allow-discrete;
}
dialog[open] {
  opacity: 1;
  scale: 1;
}
@starting-style {
  dialog[open] { opacity: 0; scale: 0.9; }
}
```

**适用场景**：Dialog/Popover 进出动画、菜单展开收起、Toast 消息。

---

### 18. `transition-behavior: allow-discrete` — 离散属性过渡

```css
.card {
  transition:
    opacity 0.3s,
    display 0.3s allow-discrete;   /* 允许对 display 做过渡 */
}

.card.hidden {
  display: none;
  opacity: 0;
}

/* 也适用于 overlay 属性（顶层元素） */
.popover {
  transition:
    opacity 0.3s,
    overlay 0.3s allow-discrete;
}
```

**适用场景**：任何 display: none ↔ block 间的平滑过渡动画。

## 📅 2025 年

### 19. Anchor Positioning（锚点定位）— 弹窗精准定位

```css
/* 定义锚点 */
.trigger {
  anchor-name: --tooltip-anchor;
}

/* 相对于锚点定位 */
.tooltip {
  position: absolute;          /* 或 fixed */
  position-anchor: --tooltip-anchor;
  top: anchor(bottom);         /* 锚点底部对齐 */
  left: anchor(center);        /* 水平居中 */
  translate: -50% 8px;         /* 微调偏移 */

  /* 自动翻转防止溢出 */
  position-try-fallbacks: flip-block, flip-inline;
}

/* 下拉菜单 */
.dropdown {
  position: absolute;
  position-anchor: --menu-btn;
  inset-area: block-end;       /* 自动在下方展开 */
}
```

**适用场景**：Tooltip、Popover、下拉菜单、上下文菜单、浮动提示。

### 20. Scroll-Driven Animations（滚动驱动动画）

```css
/* ⭐ 入场动画：元素进入视口时触发 */
@keyframes fade-in-up {
  from { opacity: 0; translate: 0 40px; }
  to { opacity: 1; translate: 0 0; }
}

.reveal {
  animation: fade-in-up 0.6s ease-out;
  animation-timeline: view();                  /* 进入视口触发 */
  animation-range: entry 0% entry 80%;         /* 滚入 0%-80% 时播放 */
}

/* ⭐ 阅读进度条 */
@keyframes progress {
  from { width: 0%; }
  to { width: 100%; }
}

.progress-bar {
  position: fixed; top: 0;
  height: 4px; background: #6366f1;
  animation: progress auto;
  animation-timeline: scroll(root);
}

/* ⭐ 视差效果 */
@keyframes parallax {
  from { translate: 0 0; }
  to { translate: 0 -100px; }
}

.parallax-bg {
  animation: parallax auto;
  animation-timeline: scroll(root);
  animation-range: entry 0% exit 100%;
}

/* ⭐ 滚动时间线监测某个元素 */
@keyframes highlight-card {
  from { outline-color: transparent; }
  to { outline-color: #6366f1; outline-width: 3px; }
}

.card {
  animation: highlight-card auto;
  animation-timeline: view(block);
  animation-range: cover 0% contain 50%;
}
```

**适用场景**：滚动视差、入场/出场动画、阅读进度条、滚动故事叙述、高亮当前阅读区域。

### 21. `light-dark()` — 一键适配明暗主题

```css
:root {
  color-scheme: light dark;
}

body {
  background: light-dark(#ffffff, #1a1a2e);
  color: light-dark(#333333, #e0e0e0);
}

.card {
  background: light-dark(#f8f9fa, #2d2d44);

.link {
  color: light-dark(#6366f1, #a5a6f6);
}
```

**适用场景**：暗色模式（无需 JS 类切换，跟随系统 `prefers-color-scheme`）。

### 22. `@scope` — 真·样式作用域

```css
/* 限定在 .card 内有效 */
@scope (.card) {
  h2 { font-size: 1.25rem; }
  p { color: #666; }
  a { color: var(--brand); }
}

/* 限定作用域并有"下界" */
@scope (.sidebar) to (.widget) {
  /* 仅 .sidebar 内的 h3（且不在 .widget 内的）生效 */
  h3 { font-size: 1rem; font-weight: 600; }
}

/* 就近优先 */
@scope (.component) {
  h2 { font-size: 1.5rem; }
}
@scope (.component.dark) {
  h2 { color: white; }
}
/* 离元素越近的作用域规则优先级越高 */
```

**适用场景**：组件样式隔离、避免 BEM 命名冲突、第三方微前端集成。

## 📅 2026 年（最新）

### 23. CSS Mixins（`@mixin` / `@apply`）— 原生混入

```css
/* 定义混入 */
@mixin --button-base {
  display: inline-flex;
  align-items: center;
  justify-content: center;
  padding: 0.5em 1.25em;
  border-radius: 6px;
  font-weight: 500;
  cursor: pointer;
  border: none;
  transition: background 0.2s, transform 0.1s;
}

/* 使用混入 */
.btn-primary {
  @apply --button-base;
  background: var(--brand);
  color: white;
  &:hover { background: color-mix(in oklch, var(--brand), black 10%); }
}

.btn-outline {
  @apply --button-base;
  background: transparent;
  border: 1px solid var(--brand);
  color: var(--brand);
}
```

**适用场景**：复用样式块、设计系统原子组件、避免重复代码。

### 25. `if()` 条件函数

.button {

**适用场景**：主题变量条件判断、变体样式切换。

### 26. `contrast-color()` — 自动对比色

```css
.button {
  background: var(--btn-bg);
  color: contrast-color(var(--btn-bg));
  /* 自动返回 black 或 white，确保 WCAG AA 合规 */
}

.badge {
  background: var(--badge-color);
  color: contrast-color(var(--badge-color));
}

/* 简化设计系统 */
:root {
  --primary: #6366f1;
  --primary-text: contrast-color(var(--primary));
}
```

**适用场景**：动态背景文字颜色、无障碍对比度保证、设计系统自动派生。

### 27. `sibling-index()` / `sibling-count()` — 兄弟计数函数

```css
/* 列表项交错动画 */
.item {
  --index: sibling-index();
  --count: sibling-count();
  --delay: calc(var(--index) * 0.1s);
  animation: fade-in 0.3s var(--delay) both;
}

/* 基于数量的布局 */
.container:has(> :nth-last-child(2)) { /* 2 个元素的特殊布局 */ }

/* 动态色环 */
.participant {
  --hue: calc(sibling-index() * 30deg);
  background: oklch(55% 0.2 var(--hue));
}
```

**适用场景**：列表交错动画、动态色环、响应式组件计数布局。

### 28. 可自定义 `<select>`（`appearance: base-select`）

```css
/* 启用自定义样式 */
select {
  appearance: base-select;
  padding: 0.5em 2em 0.5em 1em;
  border: 1px solid #d1d5db;
  border-radius: 6px;
  font-size: 1rem;
  background: white;
}

/* 自定义弹窗样式 */
select::picker(select) {
  border: 1px solid #e5e7eb;
  border-radius: 8px;
  box-shadow: 0 4px 16px rgba(0,0,0,.12);
  padding: 4px;
}

/* 自定义选项 */
select option {
  padding: 8px 12px;
  border-radius: 4px;
}
select option:checked {
  background: #eef2ff;
  font-weight: 600;
}
```

**适用场景**：美观的下拉框，不再需要 `<div>` 模拟 + JavaScript 实现。

## 🗺️ 总结：CSS 已不再只是样式语言

截至 **2026 年**，CSS 已经蜕变为一个成熟的**工程化语言**，其能力范畴覆盖：

| 类别 | 代表特性 | 解决的问题 |
|------|---------|-----------|
| **布局** | Container Queries、Subgrid、Masonry、Anchor Positioning | 组件级响应、网格对齐、瀑布流、精准定位 |
| **选择器** | `:has()`、`:is()`、`:where()`、CSS Nesting、`@scope` | 父级选择、选择器简化、作用域隔离 |
| **颜色** | OKLCH、`color-mix()`、`light-dark()`、`contrast-color()` | 感知均匀色空间、色调派生、对比度自动计算 |
| **排版** | `text-wrap: balance`、`clamp()`、逻辑属性 | 完美标题、流体排版、RTL 适配 |
| **动画** | View Transitions、Scroll-Driven Animations、`@starting-style` | 页面过渡、滚动触发动画、入场动画 |
| **代码组织** | `@layer`、`@property`、`@mixin/@apply`、`if()` | 级联分层、类型化变量、样式复用、条件逻辑 |
| **控件** | `accent-color`、自定义 `<select>`、`field-sizing` | 表单美化、原生控件自定义 |
| **响应式** | 容器查询单位（`cqw`、`cqi`）、`aspect-ratio` | 容器感知、宽高比锁定 |
| **交互** | Scroll Snap、Popover API、Anchor Positioning | 原生轮播、弹窗、工具提示 |

### 核心趋势

1. **JS → CSS 回归**：越来越多的功能（动画、布局、交互）从 JavaScript 回归 CSS 原生实现
2. **零工具链**：CSS Nesting、Mixins、`@scope` 让 Sass/Less 等预处理器不再必需
3. **感知友好**：OKLCH、`text-wrap: balance`、`contrast-color()` 更关注人的感知
4. **组件化思维**：Container Queries、`@scope`、Anchor Positioning 都围绕"组件"而非"页面"

> **一句话总结**：2020-2026 的 CSS 能力增长，超过了此前整个十年的总和。现代 CSS 可以解决过去需要 JavaScript + 预处理器 + 各种 hack 才能实现的功能。
